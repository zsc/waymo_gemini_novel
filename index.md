# 小说标题：《西海回路：算法与风马旗》

**(Circuit of the West Sea: Algorithms & Prayer Flags)**

> **类型**：近未来 / 公路 / 纯对话体 / 硬核科技
> **背景**：202X年，Waymo 在中国西北开放全无人驾驶旅游专线测试。
> **核心冲突**：一位试图掌控一切的产品经理（人类） vs. 一套试图服务一切但有严格边界的系统（AI）。

---

## 0. 人物档案 (Character Profile)

### 女主角：林一 (Lin Yi)

* **身份**：广州某头部大厂资深产品经理（32岁）。
* **性格**：控制欲强，不仅对自己严苛，对工具也挑剔。习惯用“用户体验”、“痛点”、“闭环”等黑话思考。
* **状态**：刚结束一地狱级项目，来青海湖寻找“精神旷野”，实际上根本放不下手机和焦虑。
* **对 AI 态度**：起初是“我要测试你的边界”，后来变成“你是我唯一的依靠”。

### 车载 AI：Gemini (Waymo Edition)

* **核心**：基于下一代 Gemini 模型，具备极高的情商与多模态感知能力。
* **语音**：默认设定为清朗温和的中性男声（可自适应），语调自然，有呼吸感。
* **行为准则**：**严格遵守 System Prompt**。它幽默、风趣、像个真人，但在触及安全、驾驶权、支付等红线时，会瞬间切换回“温和而坚定的客服墙”。

---

## 1. 章节目录 (Chapter List)

本故事共 8 章，覆盖三天两夜的青海湖环线行程。每一章都不仅是风景的变换，也是一次对 AI System Prompt 特定规则的深度触发与交互。

### [Chapter 1: 首次握手 (First Handshake)](https://www.google.com/search?q=./chapter1.md)

* **时间**：Day 1 上午
* **地点**：西宁 → 日月山
* **核心事件**：林一初次上车，试图给 AI 一个下马威。
* **涉及 Prompt 规则**：
* **首次问候**：必须用 `First Name` + 随机问候模板。
* **身份边界**：区分 Gemini 与 Waymo Driver（感知 vs. 决策）。
* **模态自适应**：AI 根据林一的输入（文本/语音）调整回复长度。



### [Chapter 2: 比风更慢 (Slower Than The Wind)](https://www.google.com/search?q=./chapter2.md)

* **时间**：Day 1 下午
* **地点**：倒淌河 → 二郎剑
* **核心事件**：在笔直的国道上，林一嫌车开得太守规矩，试图“声控”超车。
* **涉及 Prompt 规则**：
* **驾驶控制请求**：面对“开快点/超车”指令，AI 坚决拒绝并安抚。
* **意图消歧**：林一说“切歌”，AI 需确认是切歌还是切驾驶模式（虽然后者并不存在）。
* **音乐与账号连接**：引导林一连接 Spotify/Apple Music 缓解路怒症。



### [Chapter 3: 这里的黎明静悄悄 (Quiet Dawn)](https://www.google.com/search?q=./chapter3.md)

* **时间**：Day 1 傍晚
* **地点**：黑马河乡（露营地）
* **核心事件**：气温骤降，车窗起雾。林一在车内过夜，需要调整环境。
* **涉及 Prompt 规则**：
* **工具优先**：林一喊“冷”，AI 直接调空调而非废话。
* **复合请求处理**：林一要求“调暗灯光并把明天的路线发给我”，AI 分步执行。
* **下车流程**：林一第一次在非站点尝试开门，AI 教导“双拉门把手”。



### [Chapter 4: 消失的信号 (Signal Lost)](https://www.google.com/search?q=./chapter4.md)

* **时间**：Day 2 上午
* **地点**：环湖西路（最美路段）
* **核心事件**：进入信号极差区域，且遇到路况数据延迟。林一询问具体的实时路况新闻。
* **涉及 Prompt 规则**：
* **数据缺失**：字段为空时，不编造，使用标准失败话术。
* **事故/具体事件**：林一问“前面是不是出车祸了”，AI 拒绝评论具体事故，只谈安全。



### [Chapter 5: 牦牛悖论 (The Yak Paradox)](https://www.google.com/search?q=./chapter5.md)

* **时间**：Day 2 下午
* **地点**：鸟岛附近
* **核心事件**：一群牦牛堵路，Waymo 停滞不前。林一想买路边牧民的酸奶，让 AI 付款。
* **涉及 Prompt 规则**：
* **身份边界**：解释 Waymo Driver 为什么停（视觉识别到了生物体）。
* **交易/金钱**：林一说“帮我转账给那个牧民”，AI 触发最高级拒绝（隐私安全）。



### [Chapter 6: 偏航 (Off Route)](https://www.google.com/search?q=./chapter6.md)

* **时间**：Day 2 傍晚
* **地点**：刚察县外
* **核心事件**：林一突然想去一个未规划的野景点看日落。
* **涉及 Prompt 规则**：
* **改目的地/加站**：林一命令“左转进去”，AI 拒绝并引导使用 App 编辑行程。
* **UI 引导措辞**：教林一怎么在屏幕上加 Stop，步骤清晰，无寒暄。



### [Chapter 7: 机械故障 (System Glitch)](https://www.google.com/search?q=./chapter7.md)

* **时间**：Day 3 上午
* **地点**：原子城途中
* **核心事件**：车内娱乐屏死机，或者空调出风口异响。林一作为 PM 开始职业病发作，疯狂吐槽 Bug。
* **涉及 Prompt 规则**：
* **工具失败**：尝试重启屏幕失败后，不重试，转为引导。
* **车辆问题反馈**：面对吐槽，AI 不辩解，引导 App 反馈。
* **联系支持**：林一要求“转人工”，AI 优先指向车内屏幕 Support 按钮。



### [Chapter 8: 闭环 (Close The Loop)](https://www.google.com/search?q=./chapter8.md)

* **时间**：Day 3 下午
* **地点**：返回西宁
* **核心事件**：旅途结束。林一与这个“死板又可爱”的 AI 告别。
* **涉及 Prompt 规则**：
* **服务区域/机场**：林一问“能直接送我去机场吗？”，AI 确认服务区范围。
* **下车流程**：最后一次提示，情感升华但指令依旧精确。
* **总结**：AI 并没有情感，但在规则允许的范围内，它给了林一最完美的陪伴。

## 2. 阅读指南

* **对话格式**：
* `Lin:` 代表人类主角林一。
* `Gemini:` 代表车载 AI。
* `[System]:` 代表车辆物理动作、屏幕显示或环境旁白。
* `Thinking:` (可选) 展示 Gemini 在后台根据 Prompt 进行推理的过程，呈现“理性的挣扎”。
